# Getting started

This section explains how to use and contribute to the SMF documentation.

## Reading the documentation

Use the navigation menu or search function to find:

- `processes`;
- `equipment`;
- `safety information`;
- `projects`;
- `organizational documents`.

## [Contributing](/getting-started/contributing/)

Documentation can be edited directly through GitHub.

1. Open the relevant page.
2. Select the edit icon.
3. Make the proposed change.
4. Create a pull request.
5. Wait for review and approval.

New contributors should begin with small corrections or pages related to their own projects.
